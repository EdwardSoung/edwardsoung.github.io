프로젝트 개요

국내에서 서비스 중이던 '피구왕 통키 M'을 대만 시장에 맞춰 현지화 및 런칭하고, 약 1년 7개월간 라이브 서비스를 리딩한 프로젝트입니다. 클라이언트 파트장으로서 개발 일정 조율, 빌드 파이프라인 관리, 그리고 기술적인 이슈 해결을 주도했습니다.

Global Launch: 출시 스펙 및 빌드 관리

한국 빌드와 대만 빌드의 소스 코드를 효율적으로 분리/병합하기 위해 SVN 브랜치 전략을 수립하고, 퍼블리셔의 요구사항에 맞춰 런칭 스펙을 관리했습니다.

버전 관리 (SVN)

  • 대만 오픈 스펙에 따라 SVN 및 코드 관리.
  • 업데이트 일정에 따라 스펙 오픈.

컨텐츠 현지화

  • 대만 퍼블리셔 전용 로그인 시스템 구현.
  • 현지 유저 성향에 맞춘 이벤트 시스템 추가 개발.

Integration: 서드파티 SDK 관리

글로벌 서비스를 위해 필수적인 보안, 분석, 로그인 툴을 통합하고 안정성을 확보했습니다.

  • 보안 솔루션 (Xigncode)

    메모리 변조 및 해킹 방지를 위한 암호화 툴 적용 및 버전 대응.

  • 데이터 분석 (AppsFlyer)

    마케팅 효율 분석을 위한 트래킹 SDK 연동 및 퍼블리셔 자체 분석 툴 탑재.

Technical Challenge: AndroidX 및 API 대응

구글 플레이 스토어 정책 변경에 따라 기존 프로젝트(Android Support Library)를 최신 환경으로 마이그레이션했습니다.

Android Target API 29 → 30 상향

  • AndroidX 마이그레이션: 구형 Support 라이브러리를 사용하는 레거시 플러그인들을 AndroidX 호환 버전으로 교체 및 코드 수정.
  • 권한 처리 변경: 저장소 접근 권한 등 강화된 보안 정책에 맞춰 네이티브 연동 코드 수정.
  • Gradle 빌드 시스템 개편: 종속성(Dependencies) 충돌을 해결하기 위해 Custom Gradle Template 재구성.

Troubleshooting: SDK 충돌 해결

Issue: Facebook SDK 충돌

퍼블리셔의 자체로그인과 연동된 Facebook SDK 와 기사용중인 Facebook SDK 버전 충돌로 인한 오류 수정
