프로젝트 개요

환세취호전 온라인은 고전 명작 IP를 현대적인 모바일 MMORPG로 재해석한 프로젝트입니다. 슈퍼캣에서 팀원으로 참여하여 약 1년 1개월간 신규 컨텐츠 개발과 기존 시스템의 구조 개선(리팩토링) 업무를 수행했습니다.

특히 방대한 데이터를 다루는 업적 시스템 구축과 비효율적인 팀 편성 로직의 효율화, 그리고 환경 시스템 처리(오디오, 팝업)에 기여했습니다.

주요 업무: 컨텐츠 시스템 개발

도감 및 수집 시스템

  • 몬스터 도감: 처치한 몬스터 정보를 기록하고 보상을 지급하는 UI 및 데이터 처리 로직 구현.
  • 장비 도감: 획득한 장비의 히스토리를 관리하고 도감 버프를 활성화하는 로직 개발.

캐릭터 및 팀 관리

  • 팀 편성 버프: 특정 캐릭터 조합 시 발생하는 시너지 버프 시스템 구현.
  • 스킨 변경: 캐릭터 외형 변경 시스템 개발.

Technical: 레거시 리팩토링

기존 코드의 유지보수성을 높이고 확장성을 확보하기 위해 주요 시스템의 구조를 개선했습니다.

업적 시스템 리팩토링

신규 타입 업적을 추가하면서 기존의 타이트하게 제작된 시스템을 모듈화하여 범용성 개선

팀 편성 로직 개선

컨텐츠 별로 따로 작업되어 있던 팀 편성을 통합하여 효율적으로 개선

Technical: 코드 최적화 (LINQ 정리)

모바일 환경에서의 GC(Garbage Collection) 스파이크를 줄이기 위해, LINQ 사용을 제거했습니다.

Optimization Example

// Before: LINQ (Allocation 발생)

var activeBuffs = allBuffs
    .Where(b => b.IsActive)
    .Select(b => b.Id)
    .ToList(); // 매 호출마다 새 List 할당

// After: Loop (Zero Allocation)

_tempBuffIds.Clear(); // 재사용 리스트
for (int i = 0; i < allBuffs.Count; i++) {
    if (allBuffs[i].IsActive) {
        _tempBuffIds.Add(allBuffs[i].Id);
    }
}

▲ 편리함보다 성능이 중요한 인게임 로직에서 LINQ를 제거하여 불필요한 메모리 할당을 방지했습니다.

System: 예외 처리

게임 내 & OS 레벨의 예외처리 대응

  • 오디오 기기 변경 감지

    블루투스 이어폰 연결/해제 시 사운드 출력이 끊기거나 볼륨이 초기화되는 문제를 방지하기 위해, 오디오 장치 변경 이벤트를 감지하고 사운드 엔진을 재초기화하는 로직 추가.

  • 게임 종료 팝업 처리

    게임의 로그인 프로세스 수정에 따른 종료 프로세스 대응으로 종료 팝업 타이밍 이슈 수정.

Post Mortem

신규 개발 중인 프로젝트에 투입되어, 기존 코드의 맥락을 빠르게 파악하고 안정적으로 신규 컨텐츠를 추가하는 경험을 했습니다. 단순 구현을 넘어 "이 코드를 어떻게 고쳐야 효율적인가"에 대해 고민하며 작업을 수행했습니다.
