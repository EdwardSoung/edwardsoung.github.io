System Architecture

GameInstanceSubsystem 기반의 모듈화 및 옵저버 패턴 설계

1. GameManager

DataTable, DataAsset 데이터 파일 연결 및 보관 관리. 전역 공통 변수 제어.

// GameManager.h 
UPROPERTY(EditAnywhere, Category = "Data")
TObjectPtr<UDataTable> WidgetData;

UPROPERTY(EditAnywhere, Category = "Data")
TObjectPtr<UDataTable> PickupItemTable;

2. EventSubSystem

게임 내 모든 Delegate 통합 관리. 옵저버 패턴을 통해 컴포넌트 간 결합도 최소화.

// EventSubSystem.h 
UPROPERTY(BlueprintReadOnly, Category = "Events")
FCharacterEvents Character;

UPROPERTY(BlueprintReadOnly, Category = "Events")
FUIEvents UI;

3. UISubSystem

Lazy Initialization을 통한 위젯 생성 및 Visibility 제어. IOpenable 인터페이스 활용.

// UISubSystem.cpp 
if (CreatedWidgets.Contains(InWidgetType)) {
    Widget = CreatedWidgets[InWidgetType];
} else {
    Widget = CreateWidget(Controller, WidgetClass);
    CreatedWidgets.Add(InWidgetType, Widget);
}

4. ItemFactorySubSystem

월드 아이템 스폰 전담. 기존 아이템 데이터(수량, 내구도)를 재생성 시 그대로 유지.

// ItemFactorySubSystem.cpp 
auto SpawnClass = Manager->GetPickupItemClass(InType);
APickupItem* DropItem = GetWorld()->SpawnActor(SpawnClass, Location, ...);
DropItem->SetExistValues(Amount, InDurability);
